## Git 工作流

### 分支命名
```
main                # 主分支（生产代码）
├── dev             # 开发分支
├── feature/xxx     # 功能分支
└── fix/xxx         # 修复分支
```

### 提交流程

```bash
# 1. 从 main 创建功能分支
git checkout -b feature/add-catch-photo

# 2. 开发 + 提交
git add .
git commit -m "feat: 添加鱼货拍照上传功能"

# 3. 推送到远程
git push -u origin feature/add-catch-photo

# 4. 创建 Pull Request
# 在 GitHub 网页上创建 PR → 等待 Code Review

# 5. 合并到 dev
# 管理员合并 PR 后删除分支
```

## 小程序审核发布流程

### 1. 本地开发调试
- 使用微信开发者工具预览
- 测试号体验

### 2. 上传为体验版
```
微信开发者工具 → 上传 → 填写版本号和备注
```

### 3. 提交审核
```
微信公众平台 → 版本管理 → 提交审核
```

### 4. 审核周期
通常 1-7 个工作日（个人类小程序可能更长）

### 5. 发布
审核通过后 → 点击「发布」

## 自动化部署

GitHub Actions 自动部署后端：

1. 推送代码到 `main` 分支
2. 自动触发 `deploy.yml` workflow
3. SSH 到服务器执行 `git pull` + 重启服务

```bash
# 触发手动部署
git push origin main
```
